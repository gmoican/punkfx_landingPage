PunkFX is a audio plugin development
project created by Guillermo Moñino.



The aim of this project is getting introduced into the JUCE framework by creating a series of high-quality audio plugins that simulate the behaviour of certain analog guitar pedal effects.

I hope my work offers some insight into plugin design, real-time audio processing and guitar effects simulation.

Current state

The project is in active development. Listed plugins need some minor tweaks to be completed while many others are in their initial stage. Future additions will be:

  • Pitch shifter.
  • Envelope filter.
  • Weird bitcrusher idea.
  • Hosting plugin.

    • I hope to have finished the previous plugins before the end of the year. Once that milestone is reached, I am planning to move to other tools to study more plugin development paradigms.

    (Almost) Finished Work

    Punk Xros

    A JUCE audio plugin that pretends to emulate the behaviour of a multiband distortion pedal effect (specifically designed for bass) by means of a LSTM model.

    Punk Dist

    A JUCE audio plugin that models the behaviour of a Markbass MiniDist by means of a LSTM model.

    Punk Muff

    A JUCE audio plugin that emulates the behaviour of the Big Muff pedal effect through a LSTM model.

    Punk Komp

    A JUCE audio plugin that pretends to model the behaviour of a Suhr Koji Comp.

    Punk Boost

    A JUCE audio plugin that pretends to emulate the behaviour of a bass preamp pedal effect.

    Punk EQ

    A very simple JUCE audio plugin that pretends to model the behaviour of an equalizer pedal.

    Some final thoughs

    LSTM-based plugins

    The implementation of LSTM models for distortion simulation is fantastic in the sense that it allows for fast development and release of plugins. However, it has some drawbacks:

  • Increased CPU consumption.
  • Plugins don't build on Linux...
  • Once I've done a couple of LSTM-based plugins, I feel like I don't learn or do anything new making these plugins.

    • For these reason, I will not be doing more distortion effects under the scope of this project. I'd like to dig in electronic design in the near future, so that will be a good occassion to come back to new distiortion effects.

    What about HISE?

    JUCE is the way to go for audio plugin development (there is no avoiding that), but there are some rough aspects of building plugins with vanilla JUCE (specially if you haven't touched C++ in a long time).

    HISE is built on top of JUCE and it provides an easier experience for plugin development AFAIK. I'm saving a few ideas to have an excuse for learning this other tool one day.
